# 待机页设计需求（第一版）

## 定位

- Idle 是待机页，不是主交互页，不是第二个 Home
- Idle 的目标是让设备在无明确任务时依然有状态感、情绪感和像素生命力
- Idle 采用单焦点显示原则，同一时刻只突出一个主题
- Idle 不承载复杂操作，不承担设置入口，不承担完整通知中心

## 核心原则

- 状态驱动
- 单焦点优先
- 动效可强，但信息量要低
- 视觉氛围优先于功能密度
- 不将 Idle 做成仪表盘

## 状态模型

v1 定义四类状态：

- 默认态
- 充电态
- 媒体态
- 通知态

## 默认态

- 默认显示时间、日期、电量
- 三者视觉同级
- 构图采用中心式布局
- 默认态允许存在轻量像素背景动效
- 动效服务于氛围，不抢走信息焦点
- 默认态是所有高优先状态结束后的回落目标

## 充电态

- 充电时优先进入充电态
- 充电态核心是全屏充电动画
- 动画本身需要表达当前大致电量变化
- 时间和日期允许叠加在充电动画上层，但需要弱化
- 充电态后续允许扩展不同风格的充电动画
- v1 不要求展示预计充满时间或充电功率

## 媒体态

- 设备存在正在播放的媒体时，可以进入媒体态
- 媒体态展示封面、标题、艺人、进度、播放状态
- 封面需要像素化处理
- 媒体态只读，不提供播放控制
- 若信息不足，需要统一的像素占位方案
- 媒体态只在没有更高优先级状态时显示

## 通知态

- 新通知到达时，以全屏通知动画形式打断当前 Idle
- 多条通知按串行方式播放，不并发叠加
- 通知动画结束后，不在 Idle 中留下摘要卡或常驻痕迹
- 通知态的目标是提示发生了什么，不是让用户在 Idle 中处理通知流
- 后续可通过设置限制哪些通知有资格打断 Idle

## 状态优先级

v1 建议优先级：

1. 通知态
2. 充电态
3. 媒体态
4. 默认态

## 切换原则

- Idle 根据设备状态自动切换
- 用户后续可在设置中配置某些状态的优先级和允许显示的组件
- Idle 不要求用户主动切换页面内容
- 若某状态结束，自动回退到当前最高有效状态
- Idle 不保留复杂历史状态栈

## 夜间模式

- Idle 需要夜间模式
- 切换方式由设置决定：手动开启或按时间自动切换
- 夜间模式主要调整主题和亮度感知，不强制降低动效强度
- 夜间模式下仍保留像素氛围

## 通知与隐私

- 当前不做隐私隐藏
- v1 默认重点接入电话、短信，以及未来用户手动选择的重要 App
- 通知动画应符合“生活化、温和、符号化”的整体语气

## 设置项

- 默认态显示内容开关
- 充电动画风格选择
- 是否启用媒体态
- 是否启用通知动画
- 允许打断 Idle 的通知来源
- 状态优先级配置
- 夜间模式策略
- 动效样式或强度选项

## 不在 v1 范围内

- 完整锁屏替代
- 通知回复、媒体控制等深度交互
- 多组件同时并列显示
- 长驻通知摘要中心
- 复杂环境感知策略
- Always-on Display 级别功耗优化

## 验收标准

- 当用户没有明确任务时，Idle 能成为一个成立的待机页
- 充电时，用户一眼就能感知“正在充电”和“大致电量状态”
- 播放音乐时，Idle 能提供足够的陪伴式状态感，但不干扰
- 新通知到来时，Idle 能以明显但不嘈杂的方式提示用户
