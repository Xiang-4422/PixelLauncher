# 应用抽屉完成度与待办清单

## 文档定位

本文档只描述“当前抽屉已经落地到什么程度”，不描述理想设计目标。  
设计目标请看《应用抽屉设计需求（第一版）》，当前真实实现请同时对照 `README` 与《技术实现总览（第一版）》。

## 当前已完成

- 应用抽屉已经改成顶部对齐的纯文本列表，不再使用居中选中模型。
- 抽屉状态栏已经承担搜索入口，不再使用独立搜索框行。
- 搜索输入代理已经接通，可直接调起系统输入法进行过渡输入。
- 搜索输入已限制为 ASCII 字符，服务英文、拼音和首字母检索。
- 已支持拼音全拼匹配。
- 已支持拼音首字母匹配。
- 已支持英文别名和中文名称混合检索。
- 默认排序已实现“字母序 + 轻量 recent 修正”。
- 抽屉正文列表已接入共享文本列表基座。
- 抽屉支持惯性滚动、方向优先吸附和显式操作前的滚动收口。
- 点击应用名会直接启动，不再需要先选中再点击。
- 右侧可见字母 rail UI 已删除，改成隐藏快速定位区。
- 抽屉与 Home / Settings 的横向分页切换已经成立。

## 当前未完成

- 正式像素输入层仍未落地，当前还是系统输入法代理方案。
- 搜索态和普通态虽然已统一到同一列表基座，但搜索体验仍可继续打磨。
- 当前没有独立的搜索命中高亮表现。
- 当前没有模块级的抽屉专属设置页说明文档。

## 当前行为摘要

- 普通态：状态栏显示时间 / `SEARCH` / `APPS:当前字母`
- 搜索态：整条状态栏变成搜索栏
- 普通态可通过右侧隐藏区快速滑动定位
- 搜索态禁用隐藏快速定位区
- 列表正文允许自然裁剪，顶部和底部可以出现半行
- 抽屉列表没有可见选中高亮，但内部仍保留焦点状态
