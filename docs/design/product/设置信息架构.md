# 设置信息架构

本文档用于定义 `Settings` 模块的目标信息架构，并同步当前真实实现状态。  
阅读时请区分：

- “目标信息架构”是设置系统最终希望覆盖的范围
- “当前已实现”是代码里目前已经落地的设置页能力

当前真实实现请同时参考根目录 `README` 和 [技术实现总览](../architecture/技术实现总览.md)。

## 1. 模块定位

- Settings 是完整设置系统，不再只是显示参数面板
- 形态是单页，不做二级页面
- 目标是“可配置但不复杂”
- 当前设置页应保持像素风一致性，不像系统原生设置页贴片

## 2. 目标信息架构

目标分组顺序仍然是：

1. Display
2. Home
3. Drawer
4. Idle
5. Notifications & Data
6. Behavior
7. Advanced

这代表的是未来完整设置系统的范围，不代表当前页面已经全部落地。

## 3. 当前已实现

截至当前代码，Settings 已经落地的内容包括：

- 设置页是单页结构
- 设置页已经支持纵向滚动浏览
- 设置页正文已经接入和抽屉共享的文本列表基座
- 设置项点击已经改成直接生效，不再需要先选中再执行
- 设置页没有可见选中高亮
- 当前已落地的设置项包括：
  - `FONT`
  - `RESOLUTION`
  - `STYLE`
  - `THEME`
  - `APP ALIGN`
  - `SHOW IDLE`
  - `DRAWER SEARCH`
  - `ADVANCED`
- 当前主题中已经包含：
  - `GREEN`
  - `AMBER`
  - `ICE`
  - `MONO`
- 当前分辨率档位已经落地为：
  - `7`
  - `8`
  - `10`
  - `12`
- `ADVANCED` 已经作为 diagnostics 入口存在

## 4. 当前未实现与差距

当前 Settings 距离目标信息架构还有明显差距：

- 当前还没有真正的 `Home / Drawer / Idle / Notifications & Data / Behavior` 分组展示
- 当前还没有权限状态统一入口
- 当前还没有天气、通知、媒体、TODO 数据接入设置
- 当前还没有 Home 模块开关和卡片排序设置
- 当前还没有 Drawer 搜索细项设置
- 当前还没有 Idle 状态优先级和动效设置
- 当前 Advanced 仍然比较轻量，不是完整高级设置区

## 5. 当前实现口径

当前如果把 Settings 作为示例工程模块介绍，推荐这样表述：

- 设置页已经从“轻量参数面板”演进到“可滚动单页设置”
- 但它目前仍然主要覆盖：
  - 外观设置
  - 抽屉行为设置
  - Idle 开关
  - Diagnostics 入口
- 距离完整产品级设置系统还有较大空间

## 6. 后续演进建议

建议优先按下面顺序继续推进：

1. 先补分组结构和权限入口
2. 再补 Home / Drawer / Idle 的模块级设置
3. 最后再补 Notifications & Data 和更完整的 Advanced

## 7. 验收口径

Settings 如果要被判定为“模块成立”，至少应满足：

- 用户能在一个页面里完成主要视觉与交互偏好配置
- 设置项改动能即时生效
- 页面滚动、点击和调值行为稳定
- Advanced 不会污染主设置流
