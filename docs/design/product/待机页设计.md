# 待机页设计

本文档用于定义 `Idle` 模块的设计目标，并说明当前实现进度。  
阅读时请区分：

- “设计目标”是待机页希望最终具备的状态表达能力
- “当前已实现”是代码里目前已经成立的待机行为

当前真实实现请同时参考根目录 `README` 和 [技术实现总览](../architecture/技术实现总览.md)。

## 1. 模块定位

- Idle 是待机页，不是主交互页，不是第二个 Home
- Idle 的目标是在无明确任务时，提供状态感、情绪感和像素生命力
- Idle 应采用单焦点显示原则，同一时刻只突出一个主题
- Idle 不承载复杂操作，不承担完整通知中心

## 2. 设计目标

### 2.1 核心原则

- 状态驱动
- 单焦点优先
- 动效可强，但信息密度要低
- 视觉氛围优先于功能密度
- 不把 Idle 做成仪表盘

### 2.2 目标状态模型

v1 设计目标包括四类状态：

- 默认态
- 充电态
- 媒体态
- 通知态

### 2.3 默认态目标

- 默认显示时间、日期、电量
- 采用中心式构图
- 允许轻量像素背景动效
- 所有高优先状态结束后回落到默认态

### 2.4 充电态目标

- 充电时优先进入充电态
- 以全屏充电动画为主
- 允许弱化显示时间和日期
- 后续可扩展不同风格充电动画

### 2.5 媒体态与通知态目标

- 媒体态展示封面、标题、艺人、进度、播放状态
- 媒体态只读，不提供播放控制
- 通知态以全屏通知动画打断当前待机内容
- 多条通知串行播放，不留常驻摘要

## 3. 当前已实现

截至当前代码，Idle 已经落地的内容包括：

- 独立的 `LauncherMode.IDLE`
- Idle 进入与退出的状态切换
- `isIdlePageEnabled` 设置开关
- 基于空闲超时进入 Idle 的主流程
- `IdleFluidEngine` 背景动效
- 中心构图的时间显示
- 电池状态已经会影响待机相关表现
- 待机页已经参与主渲染链路，而不是临时独立页面

## 4. 当前未实现与差距

当前 Idle 距离设计目标还有明显差距：

- 当前默认态还没有完整显示“时间 + 日期 + 电量”三者
- 当前还没有真正独立的充电态页面内容体系
- 当前还没有媒体态
- 当前还没有通知态
- 当前没有夜间模式专属 Idle 表现
- 当前没有待机页状态优先级配置

## 5. 当前实现口径

当前如果把 Idle 作为示例工程模块介绍，推荐这样表述：

- Idle 已经具备页面状态壳层和待机物理背景
- 但当前仍然更接近“待机时钟 + 流体动效”的早期形态
- 真正的状态驱动待机系统还没有完全落地

## 6. 后续演进建议

建议优先按下面顺序继续推进：

1. 补全默认态信息结构
2. 把充电态做成真正独立的显示状态
3. 再补媒体态和通知态
4. 最后再做夜间模式和状态优先级配置

## 7. 验收口径

Idle 如果要被判定为“模块成立”，至少应满足：

- 当用户没有明确任务时，Idle 能成为一个稳定的待机页
- 充电时应能明显感知充电状态和大致电量
- 状态切换不应退化成多个组件并列堆叠
